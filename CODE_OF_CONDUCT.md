# Code of Conduct

## Purpose

AgentIgnore is a vendor-neutral open standard project. Its purpose is to help developers, tool authors, researchers, and organizations collaborate on predictable file visibility behavior for AI coding agents.

To make that possible, the project must be a place where people can discuss technical disagreements directly, honestly, and respectfully.

This Code of Conduct applies to all AgentIgnore project spaces, including GitHub issues, pull requests, discussions, documentation, RFC reviews, community calls, chat channels, and any other project-related communication.

## Expected Behavior

Participants are expected to:

* Be respectful and constructive.
* Discuss ideas, specifications, and implementation details on their technical merits.
* Assume good faith where reasonable.
* Give clear feedback without personal attacks.
* Accept criticism gracefully and improve the work.
* Prefer collaboration before conflict.
* Keep discussions focused on the project and its goals.
* Be mindful that written technical discussions can easily sound harsher than intended.
* Help newcomers understand the project without gatekeeping.

Strong technical disagreement is allowed. Harassment, intimidation, and personal abuse are not.

## Unacceptable Behavior

Unacceptable behavior includes:

* Harassment, intimidation, threats, or sustained disruption.
* Personal attacks, insults, slurs, or demeaning language.
* Discriminatory behavior or comments.
* Unwelcome sexual attention or inappropriate sexual content.
* Publishing private information without permission.
* Repeatedly derailing technical discussions.
* Bad-faith participation, including trolling or deliberately wasting maintainer time.
* Abuse of project processes, including spam, brigading, or hostile issue/PR activity.
* Retaliation against someone who reports a concern.

## Project-Specific Expectations

AgentIgnore is a standards project. Contributors should be especially careful to avoid:

* Turning vendor-neutral discussions into vendor promotion.
* Presenting personal preferences as consensus.
* Making security claims that the specification does not support.
* Using fear-based arguments instead of clear technical reasoning.
* Pressuring maintainers to accept changes without review.
* Treating disagreement with a proposal as personal rejection.

The project welcomes hard questions. Weak proposals may be rejected. That is normal technical work, not disrespect.

## Reporting

If you experience or witness unacceptable behavior, report it to the project maintainer:

* Garo Karh Bet: [garogarabed@hotmail.com](mailto:garogarabed@hotmail.com)

Reports should include:

* What happened.
* Where it happened.
* Who was involved.
* Any relevant links, screenshots, or context.
* Whether immediate action is needed.

Reports will be handled as privately as reasonably possible. Information may be shared only when necessary to investigate, take action, or comply with legal obligations.

## Enforcement

Maintainers may take any action they consider appropriate, including:

* Asking a participant to stop specific behavior.
* Editing or deleting comments.
* Locking issues or pull requests.
* Rejecting contributions.
* Temporarily limiting participation.
* Permanently banning a participant from project spaces.

Enforcement decisions should be proportional, documented where appropriate, and focused on protecting the project community.

## Maintainer Accountability

Maintainers are also bound by this Code of Conduct.

If a maintainer is involved in a reported incident, they should recuse themselves from handling that report whenever possible. As the project grows, AgentIgnore should establish a multi-person moderation and governance process.
